# Улучшение качества модели

## Текущая ситуация

Модель обучена на **87 примерах** за **3 эпохи** — это очень мало для качественного fine-tuning. Результат: модель генерирует нерелевантные ответы.

## Как улучшить качество

### 1. Увеличить датасет (КРИТИЧНО)

**Минимум**: 500-1000 примеров  
**Рекомендуется**: 5000-10000 примеров  
**Оптимально**: 50000+ примеров

Добавьте больше примеров в `example_dataset.json`:

```json
[
    {
        "instruction": "Вопрос пользователя",
        "output": "Правильный ответ модели"
    },
    ...
]
```

### 2. Увеличить количество эпох

Текущее: **3 эпохи**  
Рекомендуется: **5-10 эпох**

```bash
python fine_tuning/train.py \
    --model_name "sberbank-ai/rugpt3large_based_on_gpt2" \
    --dataset_path "example_dataset.json" \
    --output_dir "fine_tuning/lora_model" \
    --num_train_epochs 10 \
    --per_device_train_batch_size 1 \
    --gradient_accumulation_steps 8 \
    --use_4bit
```

### 3. Настроить параметры генерации

#### Более консервативная генерация (меньше "фантазий"):

```bash
python inference/chat.py \
    --base_model "sberbank-ai/rugpt3large_based_on_gpt2" \
    --lora_model "fine_tuning/lora_model" \
    --temperature 0.3 \
    --max_length 150
```

**Параметры:**
- `temperature` (0.1-1.0): чем ниже, тем более предсказуемо
  - 0.1-0.3: очень консервативно, повторяющиеся ответы
  - 0.5-0.7: баланс (по умолчанию)
  - 0.8-1.0: креативно, но может быть нерелевантно

- `max_length`: максимум новых токенов (50-300)

#### Более креативная генерация:

```bash
python inference/chat.py \
    --base_model "sberbank-ai/rugpt3large_based_on_gpt2" \
    --lora_model "fine_tuning/lora_model" \
    --temperature 0.8 \
    --max_length 200
```

### 4. Улучшить качество датасета

**Важно:**
- Примеры должны быть **разнообразными**
- Ответы должны быть **качественными** и **релевантными**
- Формат должен быть **консистентным**

**Плохой пример:**
```json
{
    "instruction": "привет",
    "output": "привет"
}
```

**Хороший пример:**
```json
{
    "instruction": "Объясни, что такое список в Python",
    "output": "Список (list) в Python — это упорядоченная изменяемая коллекция элементов. Создается с помощью квадратных скобок: my_list = [1, 2, 3]. Списки могут содержать элементы разных типов и поддерживают индексацию, срезы и множество методов для работы с данными."
}
```

### 5. Настроить гиперпараметры обучения

#### Для лучшей сходимости:

```bash
python fine_tuning/train.py \
    --model_name "sberbank-ai/rugpt3large_based_on_gpt2" \
    --dataset_path "example_dataset.json" \
    --output_dir "fine_tuning/lora_model" \
    --num_train_epochs 10 \
    --per_device_train_batch_size 2 \
    --gradient_accumulation_steps 4 \
    --learning_rate 1e-4 \
    --lora_r 32 \
    --lora_alpha 64 \
    --use_4bit
```

**Параметры:**
- `learning_rate`: скорость обучения (1e-5 до 5e-4)
  - Меньше = медленнее, но стабильнее
  - Больше = быстрее, но может переобучиться
  
- `lora_r`: rank LoRA (8, 16, 32, 64)
  - Больше = больше параметров, лучше качество, но медленнее
  
- `lora_alpha`: масштаб LoRA (обычно 2×lora_r)

### 6. Мониторинг обучения

Следите за loss:
- **Хорошо**: loss снижается плавно (10 → 5 → 2 → 1)
- **Плохо**: loss не меняется или растет
- **Переобучение**: loss на train очень низкий, но модель плохо генерирует
